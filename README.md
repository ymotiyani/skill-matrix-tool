# Skill Matrix Tool

A web-based dashboard that visualizes your organization's skill matrix data from an Excel spreadsheet. Browse employees across **9 departments**, filter by product or skill, view proficiency levels, and manage skills — all from your browser.

## Quick Start

### Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| **Python** | 3.7+ | Runs the data extraction script and local web server |
| **openpyxl** | any | Python library to parse Excel files |

Install the Python dependency:

```bash
pip install openpyxl
```

### 1. Extract data from Excel

Place your Excel file as `source.xlsx` in the project root, then run:

```bash
python extract_data.py source.xlsx
```

This parses all 9 department sheets and generates **`data.json`** — the data file consumed by the web app.

### 2. Start the local server

```bash
python -m http.server 8080 --directory .
```

> **Windows note:** Run this command from the project directory, or provide the full path:
> ```powershell
> python -m http.server 8080 --directory "C:\path\to\skill-matrix-tool"
> ```

### 3. Open in browser

Navigate to **[http://localhost:8080](http://localhost:8080)** — the dashboard loads automatically.

## Project Structure

```
skill-matrix-tool/
├── index.html          # Single-page web app (HTML + CSS + JS)
├── extract_data.py     # Python script to parse Excel → data.json
├── data.json           # Generated JSON data (do not edit manually)
├── source.xlsx         # Source Excel workbook with skill matrix data
├── audit.py            # Utility: audit/inspect extracted data per department
├── debug_sheets.py     # Utility: debug raw Excel sheet contents row-by-row
└── README.md           # This file
```

### File Details

| File | Role | When to use |
|------|------|-------------|
| **`index.html`** | The entire web application — dashboard, team pages, modals, dark mode, all in one file. Open this via the local server. | Always — this is what you view in the browser |
| **`extract_data.py`** | Reads `source.xlsx` and outputs `data.json`. Handles merged cells, multi-row employees, comma-separated skills, and 9 different sheet formats. | Run whenever the Excel data changes |
| **`data.json`** | JSON file generated by `extract_data.py`. Contains all employees, skills, proficiency levels, products, and reference data. | Auto-generated — do not hand-edit |
| **`source.xlsx`** | The Excel workbook with department sheets: Dev, QA, Cloud Ops, Design, Project Management, Product Management, Technical Writing, Product Marketing, EIT. | Provided by the team — update this with new data |
| **`audit.py`** | Prints every employee and their skills from `data.json` for quick verification. | Run to spot-check extraction results |
| **`debug_sheets.py`** | Prints raw cell values from Excel sheets to debug parsing issues. | Run when extraction produces unexpected results |

## How It Works

```
source.xlsx  ──►  extract_data.py  ──►  data.json  ──►  index.html (browser)
  (Excel)           (Python)           (JSON)           (Web App)
```

1. **`extract_data.py`** opens the Excel workbook and uses dedicated parsers for each sheet format:
   - `parse_dev_sheet()` — handles Dev's multi-category column layout (FrontEnd, .Net, Database, Cloud, Backend, etc.)
   - `parse_qa_sheet()` — handles QA's category columns (Automation, Manual Testing, API Testing, etc.)
   - `parse_cloud_sheet()` — handles Cloud Ops's category columns
   - `parse_simple_sheet()` — generic parser for the 6 newer sheets (Design, Project Management, Product Management, Technical Writing, Product Marketing, EIT), with support for merged cells and extra skill columns

2. All parsed employees are merged into a single `data.json` file.

3. **`index.html`** fetches `data.json` on load and renders the entire UI client-side — no build step, no framework, no backend.

## Features

### Dashboard
- **Stat cards** for each department showing employee counts — click any card to jump to that team's page
- Total employees, products, and unique skills at a glance

### Team Pages (9 Departments)
- **Dev** — Developers with skills across FrontEnd, .Net, Database, Cloud, Backend, Deployment, Design Patterns, Search Engines, CRM, UI Testing
- **QA** — QA Engineers with Automation, Manual Testing, API Testing, Performance Testing, CI/CD skills
- **Cloud Ops** — Cloud, Deployment, DevOps Tools, Administrator/Storage skills
- **Design** — Design team skills and proficiency
- **Project Management** — PM skills and certifications
- **Product Management** — Product management skills
- **Technical Writing** — TW Tools and Other Tools columns
- **Product Marketing** — Marketing skills
- **EIT** — Enterprise IT skills

### Employee Views
- **Card view** and **Table view** toggle
- **Search** by name or skill
- **Filter** by product (where applicable)
- Click any employee to open a **detail modal** showing all skills grouped by category with proficiency badges

### Skill Management
- **Add skills** to any employee via inline form in the detail modal
- **Remove skills** with the × button on each skill badge
- **Custom skills** — add new skills through the Skill Reference page; they become available in dropdowns
- All changes persist in **localStorage** (survives page refresh)

### Add / Remove Employees
- **Add Employee** form accessible from the sidebar
- **Remove Employee** from team pages

### UI
- **Dark mode** toggle (persisted across sessions)
- **Proficiency badges** color-coded: Expert (green), Intermediate (blue), Basic (yellow)
- Responsive sidebar navigation

## Screenshots & Walkthrough

Once the server is running at `http://localhost:8080`:

1. **Dashboard** — The landing page. Shows stat cards for all 9 departments. Click any card to navigate to that team.

2. **Team Page** — Each department page lists its members in card or table view. Use the search bar to find employees by name or skill. Use the product dropdown to filter.

3. **Employee Detail** — Click an employee card to see their full skill breakdown. Skills are grouped by category with color-coded proficiency badges. Use "Add Skill" to add new entries or × to remove.

4. **Dark Mode** — Click the 🌙 icon in the sidebar footer to toggle.

## Updating the Data

When you receive an updated Excel file:

1. Replace `source.xlsx` with the new file
2. Re-run extraction:
   ```bash
   python extract_data.py source.xlsx
   ```
3. Verify the output:
   ```
   Extracted 188 employees
     Dev: 74
     QA: 48
     Cloud Ops: 16
     Design: 8
     ...
   ```
4. Refresh the browser — the web app reads the new `data.json` automatically

> **Note:** Any employees or skills manually added/removed via the UI are stored in localStorage and overlay the JSON data. Clear localStorage to reset: open browser DevTools → Application → Local Storage → Clear.
